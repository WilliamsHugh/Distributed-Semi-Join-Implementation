BÀI TOÁN

Tại sao cần tối ưu
truy vấn phân tán?

Chi phí tổng quát

Total_Cost = CCPU + CI/O + CCOMM

Trong mạng WAN hoặc băng thông giới hạn, CCOMM thường lớn hơn hàng nghìn lần so với CI/O hay CCPU cục bộ.

Mô hình chi phí mạng

CCOMM = (Cmsg × #msgs) + (Ctr × #bytes)

Cmsg — Chi phí khởi tạo tin nhắn (Network Latency)

Ctr — Chi phí truyền tải mỗi byte (Bandwidth)

Câu hỏi nghiên cứu

Làm thế nào để giảm thiểu #bytes truyền qua mạng khi thực hiện JOIN giữa bảng Employees (10k dòng) và Assignments (50k dòng) trên 3 node phân tán?

05 / 12
PHƯƠNG PHÁP 2

Semi-Join  R ⋉ S

1
Projection tại Node 2
Chiếu EmpID, loại bỏ trùng lặp → S' (7,985 unique IDs)
2
Transfer S' → Coordinator
7,985 × 10 bytes = 79,850 bytes ✓
3
Broadcast S' → Node 1 & Node 3
Coordinator phát sóng danh sách EmpID đến cả hai node
4
Filtering song song tại Node 1 & 3
Intra-query Parallelism — CCPU giảm ½
5
Trả kết quả R'₁, R'₃ → Coordinator → Union
Chỉ gửi matched rows — ~1.25 MB tổng cộng

Tổng bytes truyền tải

1.25 MB

Size(S') + Size(R'₁) + Size(R'₃) ≈ 1,310,936 bytes

Ưu điểm

✅ Chỉ gửi khóa ngoại, không gửi cột thừa

✅ Xử lý song song tại 2 node

✅ Size Reduction Factor ≈ 75.87%

LÝ THUYẾT ÖZSU

Cost Model Analysis

Total_Cost = (Cmsg × #msgs) + (Ctr × #bytes)

Parameters

Cmsg Latency 10 ms / message
Ctr Bandwidth 0.001 ms / byte

Tại sao in-memory?

Tải CSV vào RAM → loại bỏ CI/O khỏi phương trình.
Cho phép đo lường CCOMM thuần túy.

Standard Join Comm Cost

(10 × 3) + (0.001 × 5,432,527)
= 5,462.53 ms

Semi-Join Comm Cost

(10 × 5) + (0.001 × 1,310,936)
= 1,360.94 ms

Cost reduction

−75.1%

thêm 2 message → tiết kiệm 75% bandwidth

The "Failure" Scenario

Kill Node 2 (Assignment Node) trong lúc Coordinator đang gửi request.

Kết quả mong đợi: Hệ thống timeout / catch exception thay vì treo toàn bộ (hang indefinitely).

try:
  response = httpx.get(node2_url, timeout=5.0)
except httpx.ConnectError:
  # Node 2 is down
  abort_transaction()
  return "Transaction Aborted"

Cơ chế bảo vệ

⏱

Timeout Mechanism

5 giây mỗi request

🔌

ConnectionError Catch

Phát hiện node bị sập

🛑

Transaction Abort

An toàn hủy bỏ thay vì treo

Minh chứng khả năng Fault Tolerance nền tảng trong kiến trúc Distributed Database System
